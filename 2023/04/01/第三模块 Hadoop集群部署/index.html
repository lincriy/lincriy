模块三、Hadoop集群部署

1.按照左侧虚拟机名称修改对应主机名（分别为master,slave1,slave2）,使用hostnamectl命令

master> hostnamectl set-hostname master
slave1> hostnamectl set-hostname slave1
slave2> hostnamectl set-hostname slave2

2.修改hosts文件添加左侧master,slave1,slave2节点IP，主机名映射（M,S,S）

vim /etc/hosts
ip地址 master
ip地址 slave1
ip地址 slave2
scp /etc/hosts slave1:/etc/
scp /etc/hosts slave2:/etc/

3.时区更改为上海时间(M,S,S)

rm -f /etc/localtime && ln -s /usr/share/zoneinfo/Asia/Shanghai

4.环境已安装ntp，修改master节点ntp配置，设置master为本地时间服务器，屏蔽默认server，服务器层级设为10（M）

vim /etc/ntp.conf
# 1.注释掉原来的server
# 2.添加如下内容
server 127.127.1.0
fudge 127.127.1.0 stratum 10

5.开启ntp服务（M）

systemctl start ntpd

6.添加定时任务（S,S）–在早十晚五时间段内每隔半小时同步一次本地服务器时间（24小时制，使用用户root任务调度crontab，服务器地址使用主机名）

root> crontab -e
*/30 10-17 * * * /usr/bin/ntpdate master

7.主节点生成公钥钥匙id_rsa.pub(M)

ssh-keygen

8.建立master自身使用root用户ssh访问localhost免密登录

ssh-copy-id localhost

9.建立master使用root用户到salve1免密登录

ssh-copy-id slave1

10.建立master使用root用户到slave2免密登录

ssh-copy-id slave2

11.将jdk安装包解压到/usr/java目录下（安装包存放于/usr/package277,路径自行创建，解压后文件夹默认名称，其他安装同理）

mkdir /usr/java && tar -zxf /usr/package277/jdk-8u211-linux-x64.tar.gz -C /usr/java/

12.配置系统环境变量JAVA_HOME，同时将jdk安装路径中bin目录加入PATH 系统变量，注意生效变量，查看jdk版本

vim /etc/profile
export JAVA_HOME=/usr/java/jdk-8u211-linux-x64
export PATH=$PATH:$JAVA_HOME/bin
. /etc/profile

zookeeper

1.解压zookeeper至指定目录

mkdir /usr/zookeeper && tar -zxf /usr/package277/zookeeper-3.4.14.tar.gz -C /usr/zookeeper

2.配置ZOOKEEPER_HOME,并将bin目录加入PATH系统变量

vim etc/profile
export ZOOKEEPER_HOME=/usr/zookeeper/zookeeper-3.4.14
export PATH=$PATH:$ZOOKEEPER_HOME/bin

. /etc/profile

3.修改zoo_sample.cfg为zoo.cfg

mv zoo_sample.cfg zoo.cfg

4.设置数据存储路径

dataDir = /usr/zookeeper/zookeeper-3.4.14/zkdata

5.设置日志存储路径

dataLogDir = /usr/zookeeper/zookeeper-3.4.14/zkdataog

6.设置集群列表

server.1=master:2888:3888
server.2=slave1:2888:3888
server.3=slave2:2888:3888

7.创建文件夹（数据，日志）

mkdir /usr/zookeeper/zookeeper-3.4.14/zkdata && mkdir -P zkdatalog

8.数据存储路径下写入标识主机服务器序号

echo 1 > /usr/zookeeper/zookeeper-3.4.14/zkdata/myid (M)
echo 2 > /usr/zookeeper/zookeeper-3.4.14/zkdata/myid (S)
echo 3 > /usr/zookeeper/zookeeper-3.4.14/zkdata/myid (S)

9.启动zookeeper进程，查看进程是否存在(M,S,S)

zkServer.sh start

jps

10.查看各节点服务是否正常(M,S,S)

zkServer.sh status

11.hadoop集群搭建

tar -zxf /usr/package277/hadoop-2.7.7.tar.gz -C /usr/hadoop

12.配置环境变量HADOOP_HOME,将Hadoop安装路径中bin目录加入系统变量，注意生效变量 

vim /etc/profile
export HADOOP_HOME=/usr/hadoop/hadoop-2.7.7
export PATH=$PATH:$HADOOP_HOME/bin:$HADOOP_HOME/sbin

. /etc/profile

13.配置Hadoop运行环境JAVA_HOME

vim /usr/hadoop/hadoop-2.7.7/etc/hadoop/hadoop-env.sh

export JAVA_HOME=/usr/java/jdk-8u211-linux-x64

14.设置全局参数，指定hdfs上namenode地址为master：默认端口为：9000

vim /usr/hadoop/hadoop-xx-x/etc/hadoop/core-site.xml

<configuration>
<property>
<name>fs.default.name</name>
<value>hdfs://master:9000</value>
</property>
<property>
<name>hadoop.tmp.dir</name>
<value>/root/hadoopData/tmp</value>
</property>

16.设置hdfs参数，指定备份文本数量为2

<configuration>
<property>
<name>dfs.namenode.name.dir</name>
<value>/root/hadooopData/name</value>
</property>
<property>
<name>dfs.datanode.data.dir</name>
<value>/root/hadoopData/data</value>
</property>
<property>
<name>dfs.replication</name>
<value>2</value>
</property>
<property>
<name>dfs.datanode.use.datanode.hostname</name>
<value>true</value>
</property>
<property>
<name>dfs.permissions</name>
<value>false</value>
</property>
</configuration>

17.设置yarn运行环境java_home参数

vim /usr/hadoop/hadoop-xx-x/etc/hadoop/yarn-env.sh
#some java parameters
export JAVA_HOME=/root/java/jdk-xx-x

vim yarn-site.xml
<configuration>
<property>
<name>yarn.resourcemanager.admin.address</name>
<value>master:18141</value>
</property>
<property>
<name>yarn.nodemanager.aux-services</name>
<value>mapreduce_shuffle</value>
</property>

18.设置计算框架参数，指定mr在yarn上执行

cp mapreduce-defalut.xml marped-site.xml
vim mapred-site.xml
<configuration>
<property>
<name>mapreduce.framework.name</name>
<value>yarn</value>
</property>
</configuration>

19.设置节点文件，要求master为主节点，slave1、slave2为从节点

echo -e 'slave1\nslave2' > slaves
echo 'master' > master

20.对系统进行格式化

hdfs namenode -format

21.启动hadoop，并查看各个节点

start-all.sh

22.查看集群运行状态是否正常

jps
